Primary keys

Entities must have a primary key, or identity. The title entity, for instance, has the primary key title_id, and publisher has pub_id. For both entities, the primary key is a string which must be provided when you create a new instance using CreateObject on the factory. This is known as the none idMethod just one of four identification methods Neo supports. All id-methods are:

  • none, where you have to supply the primary key,
  • native, where the primary key is generated by the database using autoincrement functionality (note that this method only works for Microsoft SQL Server),
  • guid, where a Guid is generated by Neo and used as a primary key.

You can define which id-method to use by specifying it with a idMethod attribute to the table node in you model file: 

<table name="titles" idMethod="none" javaName="Title" description="Title (Book) Table">

You can also define a global default by specifying a defaultIdMethod on the root database node: 

<database name="pubs" package="pubs.Model" defaultIdMethod="none">
