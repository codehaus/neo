What we are working on

Stability

Now that Neo is used in a major project we are discovering and fixing some intricate bugs that the tests didn't pick up. Of course, we are adding new tests for these. Xiao and Erik are also adding a suite of 'scenario tests' derived from a nexor implementation to the unit tests.

Model Diagrams

Paul is adding a new generator to the tool set that will generate output that can be transformed into an Entity Relationship Diagram using Dot.

Planned Features

Support for Oracle and other data sources

Neo, in its current form, only supports MS SQL Server. Somebody will have to write an IDataStore implementation for Oracle. The best approach seems to be to refactor the SqlDataStore implemetation such that all re-usable functionality is moved to a new superclass, DatabaseDataStore for example, and then have SqlDataStore and OracleDataStore as subclasses.

Improved qualifiers and fetch limits

Qualifiers that can contain paths and restrictions. Have a look at the following examples which should be self explanatory:

new TitleFactory().Find("Publisher.Name = 'Something'"); new TitleAuthorFactory().Find("Title[Royalty > 5000].Publisher.Country = 'Something'");

Additionally, a possibity to specify the maximum number of rows to be returned, maybe even just a boolean saying whether any row matched. For this to be meaningful it is also required to pass the sorting through to the database.

Schema generation for 'legacy' databases

Sometimes you will have the need to retrofit an object layer using Neo to an existing database. In such a case it would be cool to generate either the XML file or the object model from schema information in the database. A first step would be to add Neo support to MiddleGen.
