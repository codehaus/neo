What we are working on

Support for Enterprise Services

See the following entry Paul Gielens's blog for details: Declarative transactions at method level with Neo. (Planned for version 1.4)

Model Diagrams

Paul Hammant is adding a new generator to the tool set that will generate output that can be transformed into an Entity Relationship Diagram using Dot.

Planned Features

Improved qualifiers

Qualifiers that can contain set restrictions. Have a look at the following examples which should be self explanatory:

new TitleAuthorfactory().Find("Title[Royalty > 5000].Publisher.Country = 'Something'");

Maybe other OPath features as well. (Planned for version 1.4)

Spans

The ability to bring objects for related entities into memory in one query, e.g. when fetching publisher objecs one can instruct the context to also load all related title objects. Google for "objectspaces spans" for a discussion. (Planned for version 1.4)

Enhanced concurrency tests

See this wiki entry for details: Optimistic concurrency tests. Please provide comments if you are interested in this feature. This will also fix NEO-5. (Planned for version 1.4 or 1.5)

MultiStore

See this wiki entry for details: Using multiple data sources. Please provide comments if you are interested in this feature. (Planned for version 1.5)

Single table inheritance

The ability to define an entity classes as a subclasses of another entity class with the limitation that all data must be fetched from the same table. A classic example is a PERSON table from which one can fetch employees as well as manangers with both of them inheriting from person. (Planned for version 1.5)

Full sorting support

The current implementation only supports sorting by attributes, not paths, and the in-memory implementation is further limited to sorting by one attribute. This should be fixed. (No concrete plans)

Start with objects

See this wiki entry for details: Start with objects. Please provide comments if you are interested in this feature. (If we do it, this will be Neo version 2.)
